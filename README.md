# Логистика 365 — TMS (Transportation Management System)

Веб-приложение для управления логистикой, перевозками, транспортом и маршрутами. Построено на Django 4.2 + DRF.

> **⚠️ Это пет-проект для демонстрации.** Не предназначен для production без доработки.

## 🚀 Быстрый старт

### Локальный запуск

```bash
# 1. Активируем окружение
cd Logistic365
source venv/bin/activate  # Linux/macOS

# 2. Миграции
python manage.py migrate

# 3. Демо-данные
python manage.py seed_data

# 4. Запуск сервера
python manage.py runserver
```

Откройте http://127.0.0.1:8000/

**Учётные записи:**
| Логин | Пароль | Роль |
|-------|--------|------|
| `admin` | `admin` | Администратор |
| `dispatcher` | `dispatcher` | Диспетчер |
| `driver` | `driver` | Водитель |
| `accountant` | `accountant` | Бухгалтер |

### Docker Compose

```bash
docker compose up --build
```

Сервер: http://localhost:8000

## 🏗️ Архитектура

### Модули

| Модуль | Описание |
|--------|----------|
| **users** | Кастомная модель User с ролями (Admin, Dispatcher, Driver, Accountant). Защита от brute-force через django-axes |
| **core** | Дашборд с метриками, графиками (Chart.js), быстрыми ссылками |
| **orders** | CRUD заказов с жизненным циклом статусов, вложениями файлов, логом изменений |
| **transport** | ТС (тип, грузоподъёмность, расход топлива, статус) + водители + журнал рейсов |
| **routes** | Маршруты с точками (waypoints), расчёт расстояния/времени/топлива |
| **tracking** | События отслеживания, электронные путевые листы, вложения документов |
| **reports** | Отчёты (заказы, ТС, водители), экспорт в Excel/CSV |
| **references** | Клиенты, типы грузов, нормативы расхода топлива |

### REST API

Полный REST API доступен на `/api/v1/` с автодокументацией Swagger:

- **Swagger UI:** http://127.0.0.1:8000/api/schema/swagger/
- **ReDoc:** http://127.0.0.1:8000/api/schema/redoc/
- **OpenAPI Schema:** http://127.0.0.1:8000/api/schema/

Эндпоинты:

| Ресурс | Endpoint |
|--------|----------|
| Пользователи | `/api/v1/users/users/` |
| Заказы | `/api/v1/orders/orders/` |
| Логи статусов | `/api/v1/orders/order-status-logs/` |
| ТС | `/api/v1/transport/vehicles/` |
| Водители | `/api/v1/transport/drivers/` |
| Журнал рейсов | `/api/v1/transport/trip-logs/` |
| Маршруты | `/api/v1/routes/routes/` |
| Точки маршрута | `/api/v1/routes/waypoints/` |
| События трекинга | `/api/v1/tracking/tracking-events/` |

| Типы грузов | `/api/v1/references/cargo-types/` |
| Нормативы топлива | `/api/v1/references/fuel-norms/` |

## 📊 Функциональность

### Дашборд
- Карточки метрик (активные заказы, задержки, ТС на линии)
- Графики: распределение ТС по статусам, заказы по статусам
- Таблица последних заказов

### Заказы
- Жизненный цикл: Черновик → Подтверждён → Назначен → В пути → Загружен → Выгружен → Завершён
- Автогенерация номера заказа
- Лог изменения статусов с комментариями
- Вложения файлов (накладные, фото)
- Привязка к ТС и водителю
- Фильтрация и поиск

### Транспорт
- Учёт ТС: тип, грузоподъёмность, расход, пробег, ТО
- Статусы: исправен, в ремонте, на линии, списан
- Водители: ФИО, удостоверение, медосмотр, закреплённое ТС
- Журнал рейсов: пробег, расход топлива

### Маршруты
- Многоточечные маршруты с порядком точек
- Расчёт расстояния, времени, расхода топлива
- Leaflet-карта для визуализации

### Отслеживание
- События: прибытие, погрузка, разгрузка, выезд
- Геолокация (широта/долгота)
- Электронные путевые листы
- Вложения документов (ТТН, акты)

## 🔧 Команды управления

```bash
# Заполнить демо-данными
python manage.py seed_data --orders 50 --vehicles 15 --drivers 10

# Запуск Celery worker
celery -A logistic365 worker -l info

# Запуск Celery beat (планировщик)
celery -A logistic365 beat -l info

# Сбор статики
python manage.py collectstatic --noinput

# Запуск тестов
pytest --cov=. --cov-report=html
```

## 📋 Celery задачи

| Задача | Описание |
|--------|----------|
| `send_order_notification` | Email-уведомление об изменении заказа |
| `generate_daily_report` | Ежедневная сводка (лог) |
| `check_delayed_orders` | Проверка задержанных заказов |
| `calculate_route_statistics` | Статистика маршрутов за неделю |
| `send_telegram_alert` | Уведомление в Telegram (опционально) |

## 🔒 Безопасность

- Пароли хешированы (Django default — PBKDF2)
- Защита от brute-force (5 попыток, блокировка 1 час)
- CSRF-токены на всех формах
- Ролевая модель доступа
- Логирование действий пользователей

## 🐛 Известные ограничения (пет-проект)

- SQLite вместо PostgreSQL (для простоты разработки)
- Нет интеграции с картами (координаты вводятся вручную)
- Нет реальной интеграции с GPS-трекерами
- Нет реальной отправки email/Telegram
- Отчёты экспортируются только в HTML (Excel/CSV — заготовка)
- Нет PWA для водителей
